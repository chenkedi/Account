## ⚠️ 重要声明

本方案仅供技术研究和个人学习使用。使用前请仔细阅读以下内容：

1. **合规性**: 模拟网页API调用或浏览器自动化可能违反支付宝、微信支付等平台的《服务条款》

3. **法律风险**: 在某些司法管辖区可能涉及法律问题
4. **技术限制**: 平台反爬机制持续更新，方案可能随时失效

## 方案概述

由于支付宝、微信支付等平台**不向个人开发者开放官方API**，获取个人账户账单主要有以下技术路径：

| 方案 | 合规性 | 稳定性 | 开发成本 | 推荐度 |
|-----|-------|-------|---------|-------|
| **手动导出CSV + 导入** | ✅ 高 | ✅ 高 | 低 | ⭐⭐⭐⭐⭐ |
| **浏览器自动化** | ⚠️ 中 | ❌ 低 | 中 | ⭐⭐ |
| **模拟API调用** | ❌ 低 | ❌ 很低 | 高 | ⭐ |

## 支付宝网页版

### 1. 网页版入口
- 登录页: https://auth.alipay.com/login/index.htm
- 账单页: https://consumeprod.alipay.com/record/advanced.htm
- 我的支付宝: https://my.alipay.com/portal/i.htm

### 2. 已知的反爬机制
- **复杂的签名算法**: 请求参数包含加密签名
- **设备指纹**: 收集浏览器环境信息
- **行为分析**: 检测异常操作模式
- **验证码**: 滑块、点选等验证方式
- **Cookie有效期**: 登录态有时间限制

### 3. 网络请求分析（基于历史资料）

登录后，账单页面可能存在以下请求模式：

**请求特点**:
- URL通常包含 `consumeprod.alipay.com` 或 `mbillexprod.alipay.com`
- 请求头包含复杂的签名信息
- 参数通常经过加密处理

**可能的端点（历史记录，可能已失效）**:
- `/record/advanced.htm` - 账单页面
- `/record/tradeRecord.htm` - 交易记录

### 4. 不建议模拟API调用的原因

1. **签名算法复杂**: 支付宝使用多层加密和签名，逆向难度极大
2. **反爬严格**: 异常请求会立即触发验证或账号限制
3. **接口频繁变更**: 私有接口不定期更新，维护成本高
4. **法律风险**: 可能违反《服务条款》及相关法律法规

## 微信支付网页版

### 1. 网页版入口
- 微信商户平台: https://pay.weixin.qq.com
- 微信支付账单主要通过微信App/PC端导出

### 2. 微信PC端分析
- 微信PC端使用私有协议
- 网络通信经过加密
- 没有公开的API接口

### 3. 不建议模拟API调用的原因

1. **无公开网页版账单**: 微信支付个人账户账单主要在App内
2. **协议加密**: 微信PC端通信协议高度加密
3. **风控严格**: 微信有完善的异常检测机制

## 浏览器自动化方案

### 方案简介

使用 Playwright 或 Selenium 等工具模拟真实用户操作，实现半自动账单导出。

### 核心流程

```
1. 打开浏览器 → 2. 访问登录页 → 3. 用户扫码登录 →
4. 导航至账单页 → 5. 选择时间范围 → 6. 点击导出 →
7. 等待下载完成 → 8. 自动导入应用
```

### 技术栈推荐

- **Playwright** (推荐): 现代化、API友好、自动等待
- **Selenium**: 生态成熟、社区资源丰富
- **Puppeteer**: Chrome专用、性能较好

### Playwright 示例代码框架

```python
"""
⚠️ 仅供学习研究，使用前请确认合规性
"""
from playwright.sync_api import sync_playwright
import time
from pathlib import Path

def export_alipay_bill(output_dir: str):
    """
    半自动导出支付宝账单
    注意：需要用户手动扫码登录
    """
    with sync_playwright() as p:
        # 启动浏览器（headless=False 以便用户扫码）
        browser = p.chromium.launch(headless=False)
        context = browser.new_context(
            user_agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
        )

        # 设置下载路径
        context.set_default_timeout(60000)
        page = context.new_page()

        try:
            # 1. 打开支付宝登录页
            print("正在打开支付宝登录页...")
            page.goto("https://auth.alipay.com/login/index.htm")

            # 2. 等待用户扫码登录
            print("请使用支付宝APP扫码登录...")
            page.wait_for_url(
                "**/my.alipay.com/**",
                timeout=120000  # 2分钟超时
            )
            print("登录成功！")

            # 3. 保存登录态（可选，供后续使用）
            context.storage_state(path="alipay_state.json")
            print("登录态已保存")

            # 4. 导航至账单页面
            print("正在打开账单页面...")
            page.goto("https://consumeprod.alipay.com/record/advanced.htm")
            time.sleep(3)

            # 5. 选择时间范围（需要根据实际页面调整选择器）
            print("请手动选择时间范围后，点击导出按钮")
            print("程序将等待下载完成...")

            # 6. 等待下载
            download = page.wait_for_download()
            download_path = Path(output_dir) / download.suggested_filename
            download.save_as(download_path)

            print(f"账单已下载至: {download_path}")
            return str(download_path)

        except Exception as e:
            print(f"发生错误: {e}")
            return None
        finally:
            browser.close()


# 使用示例
if __name__ == "__main__":
    bill_file = export_alipay_bill("./downloads")
    if bill_file:
        print(f"可以将 {bill_file} 导入记账应用")
```

## 风险评估与建议

### 风险分析

| 风险类型 | 严重程度 | 说明 |
|---------|---------|------|
| **账号封禁** | 🔴 高 | 可能导致支付宝/微信账号功能受限 |
| **法律风险** | 🟡 中 | 可能违反服务条款，需自行承担责任 |
| **技术失效** | 🟠 中高 | 平台更新后方案可能随时失效 |
| **数据安全** | 🟡 中 | 需妥善处理Cookie等敏感信息 |

### 推荐方案对比

#### 方案A：纯手动导出（推荐 ✅）

**流程**:
1. 用户打开支付宝/微信App
2. 找到账单导出功能
3. 选择时间范围，导出CSV
4. 将CSV文件上传到记账应用

**优点**:
- ✅ 完全合规
- ✅ 稳定可靠
- ✅ 零开发成本
- ✅ 无账号风险

**缺点**:
- ❌ 需要用户手动操作

**改进建议**:
- 在应用内提供详细的导出步骤图文指南
- 优化CSV导入体验，支持拖拽上传

#### 方案B：半自动辅助（折中 ⚠️）

**流程**:
1. 应用提供"快速导出"按钮
2. 点击后使用 Playwright 打开浏览器并导航到账单页
3. 用户手动选择时间范围并点击导出
4. 应用自动监听下载文件夹，获取CSV后自动导入

**优点**:
- 减少部分手动操作
- 用户仍掌控关键步骤，风险较低

**缺点**:
- 仍需用户介入
- 需处理页面变化

#### 方案C：全自动化（不推荐 ❌）

**流程**:
1. 保存Cookie，实现免登录
2. 自动选择时间范围
3. 自动触发导出
4. 自动导入

**缺点**:
- ❌ 账号风险高
- ❌ 维护成本高
- ❌ 可能违规

## 对 account 项目的建议

### 短期方案（推荐）

保持现有CSV导入功能，优化用户体验：

1. **完善导出指南**:
   ```
   支付宝导出步骤：
   1. 打开支付宝App → 我的 → 账单
   2. 点击右上角"..." → 开具交易流水证明
   3. 选择"用于个人对账"
   4. 选择时间范围，申请导出
   5. 查收邮件，下载CSV文件

   微信导出步骤：
   1. 打开微信 → 我 → 服务 → 钱包
   2. 点击右上角"账单"
   3. 点击右上角"..." → 导出账单
   4. 选择时间范围，导出CSV
   ```

2. **优化导入UI**:
   - 支持拖拽上传CSV
   - 实时预览解析结果
   - 智能匹配账户

### 中期方案（可选）

如果确实需要提升自动化程度，可以考虑：

1. **浏览器辅助工具**:
   - 开发一个简易的浏览器扩展
   - 用户在浏览器中安装后，访问账单页时可一键发送数据到记账应用
   - 仍需用户手动登录，但数据传输可自动化

2. **本地文件监听**:
   - 监听系统下载文件夹
   - 检测到新的账单CSV时自动提示导入

### 长期方案（观望）

持续关注：
1. 支付宝/微信是否开放个人账单API
2. 是否有合规的第三方聚合服务
3. 银行开放银行进展

## 总结

| 方案 | 推荐度 | 说明 |
|-----|-------|------|
| **手动CSV导入** | ⭐⭐⭐⭐⭐ | 目前最稳妥的方案 |
| **Playwright半自动** | ⭐⭐ | 可尝试，但需用户知情同意 |
| **模拟API调用** | ⭐ | 不推荐，风险高且易失效 |

**最终建议**: 对于个人使用场景，**继续使用并优化现有的CSV导入功能**是最佳选择。如果确实需要自动化，建议采用**Playwright半自动方案**，并由用户完全掌控操作流程，同时明确告知相关风险。
